## 0.2.0 (2026-05-22)

0.1.x 系を trial で育てた挙動・新機能をまとめて main に取り込み、minor を一桁進めたリリース。
trial/current を `-X theirs` でマージし、その上でビルド・テスト整合を取った。

### Highlight
- `every X days from completion` 系の `rollover` 既定を **`true` → `false`** に変更（テスト整合）
- `repeat` 未設定ノートの扱いを「親切な補完」から一歩進めて、**未設定 + `next_due` なし** でも `Insert Routine` の今日候補として毎日表示するように
- 一回限り (one-off) due routine をサポート（`8568bed`）
- `tc/` ディレクトリ廃止（旧 `daily_routine.md` / `weekly_monday.md` を削除）
- v2 task grammar の整理と STATUS_PATTERN 正規表現修正
- 未来日付デイリーノートに対する操作のガード強化（[[未来日付デイリーノートとルーチン基準日ポリシー]]）
- `@done` / `→done` プロトコルと routine-atdone-marker の整備（[[start_before-@done仕様]]）
- routine-reschedule-marker の `@MMDD` / `@M/D` 等の解釈と `→YYYY-MM-DD` 正規化（[[routine-reschedule-marker仕様]]）
- iOS でデバッグ Notice がチェックボックスのクリックを吸収する問題を defer 化で解消
- ESLint 導入（warn 残あり、error なし）

### Breaking / 挙動差（既存ユーザが踏みうる）

#### 1. `every X days from completion` の rollover 既定が変わった
- これまで（バグ）: 完了起点ルーチンが overdue でも rollover=true 扱いとなり、毎日「今日」として表示され続けた。
- 0.2.0 以降: rollover=false が既定。`next_due` から `X` 日ストライドで進め、新しい `next_due` まで催促を出さない。
- 従来挙動を維持したい場合は、frontmatter に `rollover: true` を**明示**する。

#### 2. `repeat` 未指定 + `next_due` なしのノートも毎日表示候補になる
- これまで: 完了するまで表示対象に乗らないことがあった。
- 0.2.0 以降: `repeat` 未設定で `next_due` もないノートも `Insert Routine` の今日候補に並ぶ。
- 「もう出さなくていい」ノートは `repeat: none` を明示するか、`routine/` 配下から外す。

#### 3. v2 task grammar 周りの再パース
- 0.1.x 期の古い書式のタスク行が、再パース時に解釈が変わる可能性。特に時刻区切りや空行を含む行、`[[wikilink]]` と末尾見積りの組み合わせ。
- 既存デイリーノートを開いた直後に意図しない rewrite が走ったら、ログ（`llrlog/logs/debug`）で `task-parser` 系イベントを確認する。

#### 4. 未来日付デイリーノートへの操作制限
- 明日以降の daily note に対するルーチン挿入・タスク mutation がガードされる場合がある。
- 意図的に未来日に書きたい場合は、当日に切り替えるか、手動編集にフォールバックする。

#### 5. atdone / reschedule marker は metadataChanged で発火
- frontmatter を頻繁に手動編集するノートで、意図しないトリガが起きる可能性。
- `onMetadataChanged` を async 化した影響で、連続変更時に処理が並走しうる（明示ロックなし）。挙動が怪しい場合はログで `metadata-changed` 起点のイベント列を追う。

### マイグレーション要否
- ノートの書き換えは不要。
- 0.1.x で `every X days from completion` を「毎日表示」前提で運用していた場合のみ、`rollover: true` の追記を検討する。

### 互換性
- `minAppVersion`: 1.7.2（main 既存値を維持）
- `versions.json`: `0.2.0 → 1.7.2` を追加。0.1.9 のエントリも遡って追加。

### 既知の懸念 / 未対応
- 完了起点ルーチンの rollover 変更は test と挙動の意図整合は取れたが、長期運用者の感覚と微妙にズレる可能性。違和感があれば `rollover: true` で旧挙動に戻せる。
- atdone marker 処理の並走に対するロック実装は持ち越し。実害が出たら個別対応。
