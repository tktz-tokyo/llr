# Daily Note Auto-Insert Specification

## Status
- Implemented (marker-based, v0.x)
- Experimental: deferred expansion on startup / file-open for externally generated daily notes

## 1. コンセプト
新しいデイリーノートが作成されたことを検知、あるいはテンプレート内の特定のキーワードをトリガーにして、その日の予定タスク（リピートタスク等）を自動的にエディタへ書き出す。ユーザーが手動で `Insert Routine` コマンドを実行する手間を省く。

## 2. 起動トリガー

### 現行実装: テンプレート・マーカー方式（推奨）
- Obsidianの `vault.on('create', ...)` を監視する。
- ただし即挿入はせず、**新規デイリーノート内のマーカー**を検出した場合にのみ展開する。
- テンプレータ等が作成直後に内容を書き込むケースに対応するため、短時間リトライ（数秒間）を行う。

### 実験中: 後追い展開（安全寄り）
- 外部ツールや生成AIが先にデイリーノートを作るケースに備え、**今日のデイリーノートのみ**を対象に後追い展開を試みる。
- トリガーは以下。
  - Obsidian起動後の初期化完了時に1回
  - 今日のデイリーノートを開いた時
- いずれも **マーカーが残っている場合のみ** 展開する。
- `create` 起点のような短時間リトライは行わない。既存内容が安定したあとで、静かに1回だけ確認する想定。
- 展開後は `<!-- llr:routines-expanded YYYY-MM-DD -->` を残し、再実行を避ける。

### 対象ファイル
- Obsidian Core `Daily Notes` プラグインの設定（`folder`, `format`）に一致する `.md` のみ。
- それ以外の新規ノートでは動作しない。

### 対応マーカー（いずれか）
- `{{llr-today}}`
- `{{llr-routines}}`
- `<!-- llr:insert-routine -->`

## 3. 動作内容
1. デイリーノートのファイル名（日付）を `Daily Notes` の `format` で解釈し、その日付を基準日とする。
2. `RoutineEngine` を使用して、その日の日時に基づく `due` なタスクを取得。
2. 取得したタスクを `section` や時刻でソート。
3. `Routine Sections` 設定（時刻+ラベル）に基づいて `H1` 見出しへ分類する。
   - `section` 未設定は先頭（見出しなし）
   - `section` が境界時刻と同値の場合は、その見出しに入る
4. 指定されたフォーマット（`- [ ] HH:mm - [[タスク名]] (estimate)`）で文字列を生成。
5. マーカー文字列を生成したルーチンブロックで置換する（複数マーカーがあればすべて置換）。
6. マーカーが見つからない場合は何もしない（テンプレータ反映待ちのリトライを除く）。
7. 起動時 / ファイルオープン時の後追い展開は、今日のデイリーノートに限って1回ずつ安全確認する。
8. 展開済みスタンプがある場合は何もしない。

## 4. 設定項目
- v0.x では追加設定なし（マーカー固定）。
- 将来候補:
  - マーカー文字列の追加/変更
  - 自動挿入ON/OFF
  - 対象フォルダ上書き

## 5. 優先順位
- **優先度**: 中
- **ステータス**: 実装済み（v0.x）
- **補足**: 後追い展開は実験中。外部生成ノートとの共存性を見ながら条件を調整する。
