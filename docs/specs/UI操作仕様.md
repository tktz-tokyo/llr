# UI & Workflow Specification

## 0. UI Design Policy (重要)

- **ノート（メインエディタ）側の見た目は極力調整しない。**
  - LLR は、ユーザーのテーマ/Obsidian標準表示を尊重する。
  - 例外は、操作状態の視認性確保に必要な最小限の調整のみ。
  - 現在の主な例外: 実行中 `[/]` チェックボックスの視認性改善（独自表示）。
- **サイドバー（Summary View）は専用UIとして、ある程度自由にデザインしてよい。**
  - 情報の読みやすさ・操作性を優先して、レイアウト/区切り/色の調整を行う。
  - ただし、情報過多にならないように「軽い見た目」を基本方針とする。
  - 例: `summary_role: sleep` を持つルーチンは、サイドバーの `EST. TOTAL` / `EST. FINISH` 通常計算から除外し、補助表示（睡眠込み時刻）として別表示してよい。

## 1. Task Manipulation Flow

### A. Add Task (追加)
1. **Manual**: エディタで直接入力。`-[ ] [[...]]`
2. **Command**: `Llr: Insert Routine`
    - `routine/` 内の「今日期限のルーチン」を取得し、**現在開いているノートの一番下**に書き出す。
    - 書き出す際、`section` の値（`HHmm` 数値）に応じて **H3見出し** でグルーピングされる。
    - グルーピングは設定画面の `Routine Sections`（時刻 + ラベル）に従う。
      - `section` 未設定 -> 先頭（見出しなし）
      - `section >= 境界時刻` を満たす最大の見出しへ分類
      - 境界時刻と同値はその見出しに含む
    - 同一グループ内では `section` → `start` の順でソート。
    - 見出しレベルは現状 `H3` 固定。

### B. Sort Task (並べ替え)
1. **Manual**: 行の入れ替え（Alt+Up/Downなど）。
2. **Command**: `Llr: Sort by Time/Priority` (Future)
    - `sort_order` プロパティや、見積もり時間順などで並べ替える機能。

### D. Status Bar (ステータスバー常時表示)
アクティブなMarkdownノートの内容に基づき、Obsidianのステータスバーに以下の2項目を表示する。

| 表示 | 意味 | 対象タスク |
|---|---|---|
| `total: 2h35m` | ノート内の全タスクの見積もり合計 | `- [ ]` / `- [/]` / `- [x]` すべて |
| `end: 15:30` | 今の時刻 ＋ 未完了タスクの残り合計 | `- [ ]` / `- [/]` のみ |
| **`| 14:15`** | **【予定】** 今の時刻 ＋ **カーソル行のタスクを含めた**未完了タスク合計 | 行1〜カーソル行の `- [ ]` / `- [/]` |

- **残り時間の計算ロジック**:
  - **Unstarted (`- [ ]`)**: 見積もり時間をそのまま計上。
  - **Running (`- [/]`)**: `見積もり時間 - 経過時間（開始時刻から今までの時間）` を残り時間として計上（**マイナスの場合は0**）。
- **更新タイミング**:
  - エディタ内容の変更（`editor-change`）から **500ms後** に再計算（デバウンス）。
  - カーソル位置の移動（`cursor-activity`）への追従も将来的に検討。
- **対象ファイル**: 任意のMarkdownノート（`.md`）。他のファイルを開いている時は非表示。
- **見積もりの検出**:
  - **チェックボックス付き行のみ**対象（`- [ ]` / `- [/]` / `- [x]`）。
  - そのうえで、**行末** に時間表記がある場合のみ見積として採用する（例: `... 30m`, `... 1.5h`, `... 60min`, `...(30m)`, `...(30m -> 45m)`）。
  - チェックボックスのない箇条書きやログ行（例: `- skip: ...`）は集計しない。

### C. Task Status Cycle (トグル挙動)
コマンド `Llr: Toggle Task` を実行した時の状態遷移。
特定のファイル名制限はなく、Markdownファイル内の任意の行で動作する。
**制約**: カーソル行に時刻情報やチェックリストの形式が含まれない場合、新規タスクの開始とみなされる（詳細は Command Logic 参照）。

1. **Unstarted** (`- [ ]`) -> **Running** (`- [/]`)
    - 開始時刻 (`HH:mm`) を記入して `- [/]` に遷移する。
    - 見積もりがあれば `(45m)` 等を保持。
    - 開始時刻の優先順位:
      - 行頭のチェック記号直後に `HH:mm` がある場合は、その時刻を開始時刻として使う。
      - 行頭のチェック記号直後に 4桁数字 `HHmm` がある場合も、`HH:mm` と同じ開始見込み時刻として扱う。
      - 上記がない場合のみ、現在時刻で開始する。
    - 例:
      - `- [ ] 18:30 [[🥢晩ごはん]] (40m)` -> `- [/] 18:30 - [[🥢晩ごはん]] (40m)`
      - `- [ ] 1800 ばんごはん 30m` -> `- [/] 18:00 - ばんごはん (30m)`
    - 先頭時刻として解釈したトークンは、開始時刻へ昇格し、タスク本文からは取り除く。
    - 行頭以外に現れる `1300` などの時刻風トークンは、開始見込み時刻としては扱わず本文の一部として残す。
2. **Running** (`- [/]`) -> **Completed** (`- [x]`)
    - 終了時刻を記入。
    - 所要時間を計算して記入 `(42m)`。
    - **JSONLログ出力は現行仕様では非対応**（機能撤去済み）。
3. **Running** (`- [/]`) -> **Interrupt Operation** (`[x] ... ⏸` + `[ ]` duplicate) (Interrupt Command)
    - 現状は内部アクションとして実行可能で、ユーザー向けコマンド一覧には表示しない。
    - 現在時刻で完了ログ (`[x]`) を確定し、末尾に `⏸` を付与。
    - 同時に未着手タスク (`[ ]`) を直下に複製して再開待ちにする。
    - 見積りがある場合は残り見積りを再計算する。

### E. Checkbox Press Override (編集モード)
**Status: Implemented**

編集モードでは、チェックボックス操作を Obsidian 既定挙動から LLR 挙動へ上書きする。
適用範囲は **モバイル/デスクトップ共通**。
設定 `Editor Checkbox Override` が ON のときのみ有効で、OFF のときはクリック挙動だけ Obsidian 標準に戻る。コマンドとショートカットは常に利用可能。

- **短押し**:
  - `- [ ]` -> `- [/]`
  - 開始時刻は通常トグルと同じルールを使う。
  - つまり、チェック記号直後の先頭トークンが `HH:mm` / `HHmm` ならそれを開始時刻に使い、なければ現在時刻を使う。
  - `- [/]` -> `- [x]`（完了）
- **長押し（目安: モバイル 450ms / デスクトップ 900ms）**:
  - `- [ ]` -> `- [/]`（直近完了タスクの終了時刻で開始。見つからない場合は現在時刻）
  - `- [/]` -> `- [ ]`（未着手へ戻す。未着手として再解釈可能な形に正規化し、開始時刻1つ・見積りを保持）
  - `- [x]` -> `- [ ]`（未着手へ戻す。未着手として再解釈可能な形に正規化し、実績時刻は落として見積りを保持）

- **未着手行の先頭時刻トークン解釈**:
  - 対象は `- [ ]` の直後、本文の**先頭トークンのみ**。
  - `18:00` と `1800` は同じ意味で扱う。
  - サイドバーやサマリー表示でも同じルールを使い、時刻トークンは本文表示から除く。
  - `- [ ] 1800 ばんごはん 30m` は、内部的に `開始見込み: 18:00 / 本文: ばんごはん / 見積り: 30m` として扱う。

- **タップ判定の厳格化 (Precision Targeting)**:
  - チェックボックス要素（アイコンおよびその直近のコンテナ）へのタップのみをトグル対象とする。
  - **テキスト部分（タスク名など）へのタップは無視し、Obsidian標準のカーソル移動挙動を優先する。**
  - **適用範囲の限定**: オーバーライドは**Markdown編集ビュー（Editor）内部の要素**に対してのみ適用される。設定画面（Settings）、サイドバー、モーダルなどのObsidianコアUIや他プラグインのUI要素には一切干渉せず、既定のイベント伝播を維持しなければならない。
- **操作対象行の確定**:
  - トグル操作は必ず「タップされたチェックボックスが存在する行」に対して行われる。
  - **カーソルが別の行にある場合でも、タップされた行が優先される。** 行番号の解決に失敗した場合は、誤動作防止のため処理を中断する（カーソル行へのフォールバックは行わない）。
- 長押し成立後の後続クリックは抑止し、二重実行を防ぐ。
- **実装上の制約**: 
  - イベントリスナーを `capture: true` で登録する場合、要素判定を `getCheckboxAtPoint` 等で厳格に行い、LLR の管理外（エディタ外）であると判断された場合は即座に関数を抜け、イベントの伝播やデフォルト動作を妨げてはならない。
- コマンド体系は拡張済み:
  - `Start Task at Previous Time`（前の時刻で開始）
  - `Reset Task (Keep Estimate)`（時刻保持で未着手に戻す）
- 誤タップUndoや設定UIは次フェーズで検討。
- モバイルでは対応端末で触覚フィードバックを行う（短押し: 1回、長押し: 2連続）。

### 旧仕様メモ（廃止）
- 以前は `Interrupt` により `- [-]`（中断ステータス）を生成していたが、現行仕様では廃止。

## 2. Date Boundary (Log Rotation)
**Status: Not Implemented**

### Command: `Llr: Start New Day`
1. **Check**: 現在のデイリーノート/作業ノートが空でないか確認。
2. **Archive**: ファイル名を `llrlog/archive/YYYY-MM-DD_llr.md` に変更して移動。
3. **Rollover**:
    - アーカイブされたファイルから `Unstarted` なタスクを抽出。
    - 今日の `Routine` (daily + 曜日一致) を抽出。
    - 新しい当日ノートを作成し、それらを書き込む。

この際、旧ファイルへのリンク切れを防ぐため、アーカイブ作成は「移動(Rename)」ではなく「コピー＆クリア」方式を採用するか、Obsidianのリンク追従に任せるかを実装時に決定する。
