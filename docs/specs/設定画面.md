# Configuration Specification

**Status: Partially Implemented**

プラグイン設定画面で提供するオプションの仕様。現状実装と将来候補を分けて記載する。

## 1. Implemented Settings (Current)

| Key | Type | Default | Description |
|---|---|---|---|
| `Estimate Warning` | boolean | `true` | 残り見積り時間に基づく予定警告の表示を切り替える。 |
| `Routine Folder` | string | `routine` | ルーチンノート格納フォルダ。サジェスト候補から選択できる。 |
| `Routine Sections` | list (`label`,`time`) | `午前/0700`, `午後/1200`, `夜/1800` | `Insert Routine` とデイリーノートのマーカー展開時の見出し区切り。サイドバーの section divider 表示にも利用。 |

## 1.1 Advanced / Compatibility

| Key | Type | Default | Description |
|---|---|---|---|
| `Debug mode` | boolean | `false` | デバッグ用 `Notice` と JSONL トレースログを有効化する。詳細は [[デバッグモード]]。 |
| `Editor Checkbox Override` | boolean | `true` | 編集画面のチェックボックスクリックを LLR の短押し/長押し挙動で上書きする。OFF のときは Obsidian 標準クリックに戻り、コマンド/ショートカットはそのまま使える。 |

## 2.1 Routine Folder UI (Implemented)

- 入力欄はフォルダ候補サジェストを表示する。
- 候補選択時（タップ/クリック）は、選択した `folder.path` をそのまま保存する。
- `Enter` または `blur` で確定する場合は、入力文字列を次の順で解決して保存する。
  - 完全一致する実在フォルダ
  - 先頭候補（入力文字列で始まる候補のみ）
- 実在フォルダに解決できない中途値は保存しない（現在の設定値へ戻す）。

## 3. Routine Sections UI (Implemented)

### 3.1 Data Model
各項目は以下の2値を持つ。
- `label`: 見出しラベル（`H3`本文）
- `time`: 4桁時刻 `HHmm`

### 3.2 編集UIの挙動
- 既存行は `ラベル -> 時刻` の順で編集する。
- `New Section` 用の未確定入力行を常設する（押した瞬間に行が飛ばないようにする）。
- 追加確定条件（いずれでも可）:
  - `Enter`
  - `blur`（フォーカス外れ）
  - `+` ボタン
- **ラベルと時刻の両方が埋まっている時のみ**追加確定する。
- 保存時に **時刻順で自動ソート**する。

### 3.3 Validation / Classification
- 無効な時刻（`HHmm` 以外、または `24:00` 等の不正値）は保存しない。
- 分類ルール（`Insert Routine` / 自動挿入）:
  - ルーチンノートの `section` 未設定 -> 先頭（見出しなし）
  - `section >= time` を満たす **最大の時刻** のラベルへ入る
  - 境界時刻と同値はその見出しに含む

## 5. Notes
- 現状の設定は `loadData/saveData` に保存される。
- `Debug mode` は通常運用ではOFFを推奨（通知/ログ書き込みが増えるため）。
