# Cheat Sheet (カンペ)

## 0. こんな時 → これ（逆引き）

### 表示まわり
| やりたいこと | やり方 |
|---|---|
| このルーチンを今日だけ先送りしたい | `Reschedule Routine` か、タスク行末に `@MMDD`（例: `@0530`）を書いて保存 |
| 「もう今回ぶんは済ませた」と前倒しで閉じたい | タスク行末に `@done` を書く（`start_before` 必須） |
| 完了してないルーチンを翌日も出し続けたい | ルーチンノートの frontmatter に `rollover: true` |
| `next_due` を過ぎたら自動で次の期日に進めたい | `rollover: false`（既定なので何も書かなくて良い） |
| サイドバー / 集計から睡眠タスクを除外したい | ルーチンノートに `summary_role: sleep` |
| このルーチンを停止したい | `repeat: none`（または `0` / `no`） |
| `next_due` の数日前から表示しておきたい | `start_before: 3`（3日前から表示） |
| ルーチンを一回限り（来週だけ）出したい | `repeat: none` のまま `next_due: 2026-05-29` を書く |

### 操作まわり
| やりたいこと | やり方 |
|---|---|
| 開始時刻を1分早めたい | `Adjust Time (1m)`（時刻1つなら開始 -1, 2つなら終了 -1） |
| 一度中断してまた続きをやりたい | `Interrupt Task`（完了行＋未着手の続き行を生成） |
| 手動で打った完了行の duration がズレた | `Fix Duration Drift (All Completed Tasks)` |
| 後から「これ実は完了してた」を記録したい | `Retro Complete Task` |
| 既存タスクをそのまま複製したい | `Duplicate Task` |
| 未着手に戻したい（見積りは残す） | `Reset Task (Keep Estimate)` |
| ログだけ残してこのタスクは消したい | `Skip Task (Log Only)` |

### 「できる?」の即答用
| 質問 | 答え |
|---|---|
| タイマーや通知を鳴らしたい | できません（範囲外。`docs/specs/STATE_実装状況サマリー.md` §3）|
| カレンダー連携したい | できません（範囲外） |
| グラフや統計を表示したい | できません（範囲外。記録正本は Markdown） |
| サイドバーでタスクを直接編集したい | まだできません（未実装。`STATE` §2.B） |
| ルーチンフォルダを変えたい | 現状 `routine/` 固定（設定UI 未実装） |

詳細はそれぞれの仕様書を参照。範囲外の正本は [[specs/STATE_実装状況サマリー]] §3。

## 1. `repeat` の書き方（私の推奨）

`routine/` 配下ノートの YAML Frontmatter に書く。

```yaml
---
repeat: 1
next_due: 2026-02-26
---
```

### A. 毎日・日数間隔（数字だけ）

- `1`（毎日 / 基準日の翌日）
- `5`（5日ごと / 基準日から5日後）
- `10`（10日ごと / 基準日から10日後）
- `3日後`（完了日から3日後）
- `10日後`（完了日から10日後）
- `0`（繰り返ししない / 停止）

### B. 曜日指定

- `月水金`（毎週 月・水・金）
- `毎週月曜`（毎週 月曜）
- `隔週月曜`（2週ごと 月曜）

### C. 月次

- `毎月1日`
- `毎月5,10,15日`
- `毎月末`
- `毎月末-1日`（月末の1日前）
- `第2土曜日`（毎月第2土曜）
- `最終土曜日`（毎月最終土曜）

### D. 迷ったらこれ

- 毎日: `1`
- 日数指定: `3日後`
- 平日ルーチンっぽいもの: `月火水木金`
- 週1: `毎週月曜`
- 月1: `毎月1日`
- 停止: `0`

### E. どこに書くか（例）

- YAML Frontmatter の `repeat:` に入れる
- 例:

```yaml
---
repeat: 月水金
next_due: 2026-02-26
---
```

### F. 詳しい書き方

- 詳細仕様: [ルーチンノートFrontmatter仕様](specs/ルーチンノートFrontmatter仕様.md)

## 2. チェックボックス操作（編集モード）

### 短押し

- `- [ ]` -> `- [/]`（開始）
- `- [/]` -> `- [x]`（完了）
- `- [x]` -> 何もしない

### 長押し（モバイル目安 450ms / デスクトップ目安 900ms）

- `- [ ]` -> `- [/]`
  - 近くの完了行の終了時刻で開始（上を優先、なければ下も見る）
  - 見つからなければ現在時刻
- `- [/]` -> `- [ ]`（未着手へ戻す。未着手として再パース可能な形に正規化し、開始時刻1つ＋見積り保持）
- `- [x]` -> `- [ ]`（未着手へ戻す。実績時刻は落とし、見積りを保持）

## 3. よく使うコマンド（実用順）

### 日常の2つ

- `Toggle Task`
- `Adjust Time (1m)`（スマートに1分戻す）
  - 時刻1つ: 開始を -1分
  - 時刻2つ以上: 終了を -1分
  - 完了行の duration は再計算

### 補助

- `Start Task (Align to Previous Completion)`（前に合わせる）
- `Reset Task (Keep Estimate)`（未着手として再パース可能な形に戻す / 見積り優先）
- `Duplicate Task`（未着手で複製 / 自動開始しない）
- `Interrupt Task`（完了行 + 続きの未着手行を作る）
- `Retro Complete Task`（後追い完了を明示実行）
- `Fix Duration Drift (All Completed Tasks)`（完了行 duration の一括補正）
- `Skip Task (Log Only)`（未着手行と `skip` ログを相互変換）

## 4. 行フォーマットの早見

- 未着手: `- [ ] タスク (30m)`
- 実行中: `- [/] 09:00 - タスク`
- 完了: `- [x] 09:00 - 09:28 (28m) タスク`
- 完了（見積差分あり）: `- [x] 09:00 - 09:28 (30m -> 28m) タスク`

## 5. 困った時の確認

- 完了行の文言を編集しただけで `next_due` が動かないか（デバッグログ）
- `Adjust Time (1m)` が見当たらない時はプラグイン再読み込み
- モバイルは `Toggle Task` + `Adjust Time (1m)` を先に配置
